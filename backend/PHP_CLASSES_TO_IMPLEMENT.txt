================================================================================
TechTrade SA — PHP REST API Class Structure
Suggested namespaces and responsibilities (PSR-4 autoloading)
================================================================================

Recommended stack
-----------------
- PHP 8.2+
- Composer + PSR-4 (e.g. namespace App\)
- Slim 4 / Laravel API / vanilla router — patterns below work for any
- PDO for MySQL (schema: database/schema.sql)
- vlucas/phpdotenv for config
- firebase/php-jwt OR native PHP sessions for auth

Suggested folder layout
-----------------------
backend/
  public/index.php          # front controller
  src/
    Config/
    Http/
    Middleware/
    Controllers/
    Services/
    Repositories/
    Models/
    DTOs/
    Validators/
    Exceptions/
    Integrations/
  routes/api.php
  config/
  .env

================================================================================
1. CONFIG
================================================================================

App\Config\DatabaseConfig
  - DSN, credentials, PDO options (ERRMODE_EXCEPTION, FETCH_ASSOC)

App\Config\AppConfig
  - app env, debug, base URL, CORS allowed origins, cookie domain

App\Config\AuthConfig
  - JWT secret / session name, TTLs, bcrypt cost

App\Config\PayFastConfig
  - merchant_id, merchant_key (env only), sandbox flag, notify URL

App\Config\StorageConfig
  - listing/dispute image paths, S3/local, max upload size

================================================================================
2. HTTP LAYER (request / response)
================================================================================

App\Http\Router
  - register routes, method + path → controller action

App\Http\Request
  - wrap $_SERVER, headers, JSON body, authenticated user id

App\Http\Response
  - json(), error(), status codes, CORS headers

App\Http\JsonBodyParser
  - parse application/json for POST/PUT/PATCH

================================================================================
3. MIDDLEWARE (run before controllers)
================================================================================

App\Middleware\CorsMiddleware
  - Allow-Origin, Allow-Credentials, preflight OPTIONS

App\Middleware\JsonMiddleware
  - Content-Type application/json

App\Middleware\AuthMiddleware
  - Validate session or JWT; attach User to request

App\Middleware\RoleMiddleware
  - Require role: buyer | seller | admin | moderator

App\Middleware\CsrfMiddleware
  - Validate tt_csrf cookie vs X-CSRF-Token header

App\Middleware\RateLimitMiddleware
  - Per-IP / per-user limits (login, OTP, reports)

App\Middleware\TransactionParticipantMiddleware
  - Ensure current user is buyer or seller on transaction id

App\Middleware\ListingOwnerMiddleware
  - Ensure current user owns listing

5. SERVICES (business logic)
================================================================================

App\Services\AuthService
  - register, login, logout, password reset, OTP issue/verify

App\Services\UserService
  - profile update, trust score recalc, avatar upload

App\Services\ListingService
  - create/publish listing, status transitions, featured flag

App\Services\MarketplaceService
  - filter listings (category, price, condition, province)

App\Services\OfferService
  - create offer, accept/decline, counter-offer chain

App\Services\TransactionService
  - create transaction from offer, escrow step machine (1–5)

App\Services\EscrowService
  - advance step rules (buyer vs seller actions), 48h inspection window

App\Services\PaymentService
  - build PayFast signature, record payment, handle ITN webhook

App\Services\DisputeService
  - open dispute, freeze escrow, attach evidence (max 6 images)

App\Services\ReviewService
  - post-transaction reviews, update user trust_score

App\Services\ReportService
  - user/listing reports → admin queue

App\Services\WhatsAppTTLService
  - save user preset (24h|7d|90d|custom + days), validate custom 1–365

App\Services\WhatsAppBridgeService
  - create bridge on transaction commit, compute expires_at from user preset
  - purge wa_deeplink / bridge_token on expire; write audit_metadata_json only

App\Services\NotificationService
  - create notifications (offer, counter, bridge), mark read

App\Services\BuildDoctorService
  - compatibility / PSU wattage logic (can call rules or external API)

App\Services\FileUploadService
  - validate MIME/size, store listing + dispute images

App\Services\Admin\AdminReportQueueService
App\Services\Admin\AdminUserModerationService
